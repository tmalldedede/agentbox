# AgentBox 商业计划书

> **一句话定位：开源版 Manus**

---

## 1. 项目概述

### 1.1 什么是 AgentBox

AgentBox 是一个 **开源的 AI Agent 云交付平台**，让企业和开发者能够轻松部署、管理和运行专业 AI Agent（如 Claude Code、Codex、OpenCode）。

### 1.2 核心价值主张

| 对比 | Manus | AgentBox |
|------|-------|----------|
| **开源** | ❌ 闭源 SaaS | ✅ 完全开源 |
| **私有部署** | ❌ 仅云服务 | ✅ 支持自托管 |
| **数据安全** | 数据上云 | 数据本地化 |
| **Agent 选择** | 仅 Manus Agent | 多 Agent（Claude/Codex/OpenCode） |
| **可定制** | 黑盒 | 完全可控 |
| **成本** | 高昂订阅费 | 免费 + 可选服务 |

### 1.3 为什么是"开源版 Manus"

- **Manus 的成功验证了市场需求** —— AI Agent 执行复杂任务
- **但企业有顾虑** —— 数据安全、成本、厂商锁定
- **AgentBox 填补空白** —— 同样能力，开源可控

## 2. 市场分析

### 2.1 目标市场

```
AI Agent 基础设施市场
├── 企业自动化（代码生成、文档处理、数据分析）
├── DevOps 集成（CI/CD 中的 AI 任务）
├── AI 应用平台（需要 Agent 执行能力的 SaaS）
└── 安全敏感行业（金融、医疗、政府）
```

### 2.2 市场规模

| 年份 | 全球 AI Agent 市场规模 | 增长率 |
|------|----------------------|--------|
| 2024 | $5.2B | - |
| 2025 | $8.1B | 56% |
| 2027 | $18.5B | 51% |
| 2030 | $47.1B | 37% |

*来源：Grand View Research, MarketsandMarkets*

### 2.3 竞争格局

| 竞品 | 定位 | 优势 | 劣势 |
|------|------|------|------|
| **Manus** | 闭源 AI Agent | 产品完善、体验好 | 闭源、贵、数据上云 |
| **E2B** | 代码沙箱 | 融资多、Firecracker | 闭源、仅沙箱 |
| **Modal** | AI 计算平台 | 技术强、gVisor | 不专注 Agent |
| **Daytona** | Dev 环境 | 开源 | 不是 Agent 平台 |
| **AgentBox** | 开源 Agent 平台 | 开源、多 Agent、Profile 系统 | 早期、需打磨 |

### 2.4 差异化竞争力

1. **完全开源** —— 唯一开源的 Manus 替代品
2. **Agent 无关** —— 不锁定单一 Agent，支持 Claude/Codex/OpenCode
3. **Profile 系统** —— 可复用、可继承、可市场化的配置模板
4. **私有部署** —— 满足数据合规需求

## 3. 产品架构

### 3.1 核心概念

```
┌─────────────────────────────────────────────────────────┐
│                    AgentBox 架构                         │
├─────────────────────────────────────────────────────────┤
│                                                         │
│   用户/应用  ──→  Task API  ──→  AgentBox 平台          │
│                                                         │
│   ┌─────────────────────────────────────────────────┐   │
│   │                  Profile 系统                    │   │
│   │   Profile = Agent + MCP + Skills + Credentials  │   │
│   │              + 权限 + 资源限制                    │   │
│   └─────────────────────────────────────────────────┘   │
│                          ↓                              │
│   ┌─────────────────────────────────────────────────┐   │
│   │               Container Runtime                  │   │
│   │   ┌─────────┐ ┌─────────┐ ┌─────────────┐       │   │
│   │   │ Claude  │ │ Codex   │ │  OpenCode   │       │   │
│   │   │  Code   │ │         │ │             │       │   │
│   │   └─────────┘ └─────────┘ └─────────────┘       │   │
│   └─────────────────────────────────────────────────┘   │
│                                                         │
└─────────────────────────────────────────────────────────┘
```

### 3.2 核心功能

| 功能 | 说明 | 状态 |
|------|------|------|
| **多 Agent 支持** | Claude Code、Codex、OpenCode | ✅ 已实现 |
| **Profile 系统** | 配置模板、继承、复用 | ✅ 已实现 |
| **MCP 集成** | 扩展 Agent 工具能力 | ✅ 已实现 |
| **Skill 系统** | 预定义任务模板 | ✅ 已实现 |
| **Credential 管理** | 安全存储 API Key | ✅ 已实现 |
| **容器隔离** | Docker 容器运行 | ✅ 已实现 |
| **Task API** | 对外服务接口 | ✅ 已实现 |
| **Web 管理界面** | React 前端 | ✅ 已实现 |
| **任务队列** | 调度、重试 | 🔄 进行中 |
| **多租户** | 租户隔离、配额 | 📋 计划中 |

### 3.3 技术栈

- **后端**：Go + Gin + Docker SDK
- **前端**：React + TypeScript + Tailwind CSS
- **存储**：SQLite（单机）/ PostgreSQL（集群）
- **容器**：Docker / Kubernetes
